Game Generation Engine (how each game avoids repetition)

Structural Dynamism (Phase 1+2)

5 predictability vectors that any game can leak. We rotate each, not just stimulus values:

  • V1 Target position — random within display (Flanker pos 1–5, Corsi 6/9/12/16 cells, Trail 1–25)
  • V2 Response set — modality rotates (tap / swipe / hold / multi-tap / keypad)
  • V3 Rule semantics — same stimulus, different response meaning (color / length / count)
  • V4 Cadence — ITI jittered 800–2000ms, no rhythm learning
  • V5 Feedback grammar — commission feedback only on errors, no green-check treadmill

Anti-Exploitation Distribution Drift

Detects when the user has learned a structure and shifts the distribution. Examples:

  • Position bias: if user RT on position-3 is <50% of position-1 RT, bias next 10 trials toward position-1
  • Response bias: if user is on a right-tap streak, inject 70% left-tap block
  • Strategy detection: if Corsi taps follow a path (always L→R), break the path
  • Rhythm detection: if ITI-resp RT has CV < 0.15, add ±400ms jitter

Catch Trials & Rule Injections

Hidden rule changes that the user must detect and adapt to. No explicit "Round 2" label.

  • 8% of trials in Context Switcher invert the rule (word=ink briefly becomes ink=word)
  • Pure-num blocks injected in Trail Making B (3 trials of just 1-2-3 between alternating blocks)
  • Obstacles appear mid-task in Zoo Map (re-planning demand)
  • Constraint shifts in Verbal Fluency (animals → verbs → 4-letter words) without announcement
  • Symbol rotation in Cancellation every 3 sessions

Procedural Generation & Scoring (preserved from Phase 0)

Procedural Trial Generation

Every game uses a generator function, not a fixed item pool. Examples:

  • Digit Span: Random digit sequences with constraints (no repeats >2, no ascending runs >3)
  • Mental Arithmetic: Parameterized equation constructor (a ± b = c, difficulty = digit length)
  • Flanker: Random arrow direction + randomized target position + congruent/incongruent flanker assignment
  • Word List: Sampled from 3,000+ word pool, frequency-banded, balanced by syllable count
  • Reading Passages: Rotating pool of 500+ curated passages, questions generated from templates

Adaptive Difficulty (Trial-Level)

Each game's difficulty engine is unique to prevent "cracking" the progression:

  • Staircase: 1-up/1-down (standard threshold estimation)
  • Bayesian: Real-time theta estimation (IRT-based)
  • Weighted: Difficult = target accuracy 70%, Easy = 90%
  • Blocked: Fixed sequence of pre-calibrated difficulty levels
  • Mixed: Interleaved easy/hard for engagement + measurement
